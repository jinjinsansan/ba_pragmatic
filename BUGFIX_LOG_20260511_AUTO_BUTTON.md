# BUGFIX LOG — AUTOボタン改修・安定化（2026-05-11）

## 背景
- 症状1: AUTO押下後に BET が出ない／止まることがある
- 症状2: 手動は「BET窓が光った瞬間」なら成功しやすいが、AUTOは stale が多い
- 症状3: AUTOボタンの押下感が弱く、手動ボタンより操作フィードバックが少ない
- 症状4: AUTO時の結果表示が手動と比べて弱く、失敗理由の可視性が不足

## ログ調査で確認した事実
- `auto_bet` の失敗主因は `decision_stale`（queue_delay が 18〜23 秒帯で多発）
- 一部 `bet_window_closed` も発生
- 受け子ログで `SIGNAL BET` は到達しており、クリック未到達ではなく「送信タイミングと滞留」が主因
- ロビー再遷移ログ（`[lobby] clicked ...`）が増える副作用を確認

## 実装した修正（master UI）
### 1) AUTO送信タイミングを手動代理モデルへ寄せた
- BET窓の「OPEN立ち上がりエッジ」で送信するように変更
- `pending/processing` の `auto_bet` が残っている間は次送信しない制御を追加
- 前回失敗で反映されない場合のみ、次の OPEN エッジで再送許可

### 2) AUTOボタンのUI強化
- 立体感（膨らみ）を付与
- 押下時に凹むアニメーションを強化（:active + インナー演出）
- hover時の浮き上がり・発光を追加

### 3) AUTO結果表示を手動系に統一
- `lastActionBox` に AUTO 送信中／受理／失敗／通信エラーを表示
- 停止中・テーブル未選択時も明示的にエラー表示

## 途中で発生した不具合と対応
- 一時的に JS 構文エラーを混入し、ハードリロード後に「読込中のまま」になった
- 該当行を即修正し再デプロイ、`/master` 応答正常を確認して復旧

## デプロイ/確認
- `bacopy_master_ui.py` を VPS (`/opt/bacopy/bacopy_master_ui.py`) へ反映
- `bacopy-api` 再起動後 `active` を確認
- ローカルとVPSのファイルハッシュ一致を確認

## 現在の見立て
- 「完全停止」は解消方向
- ただし実運用では引き続き `decision_stale` 監視を継続し、必要なら executor 側 stale 条件・待機閾値の再調整を行う
