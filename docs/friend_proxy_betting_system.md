2026-04-17 に構想化。Evolution/Pragmatic で 52%勝率を出している友人の判断を、GUI + 機械で執行 → 決定ログを蓄積 → AI が学習 → 友人不在でも代理判断、というハイブリッドシステム。

**Why:** Evolution/Pragmatic の自動化戦略は全てエッジなし (2026-04-15~17 の徹底検証で証明)。house edge が方向を問わず抽出される設計。一方で人間 (友人) は 52%勝率 = 真のエッジを持っている。**機械に勝てるロジックを作るのではなく、勝てる人間の判断を機械でスケールさせる**方が理にかなう。

**How to apply:** GUI 設計の起点として参照。決定ロガー機能は**初期実装時から必須**。商用配布ではなく「友人の知的財産を活用する補助ツール」というビジネスモデル。

## 基本思想

- **機械の勝てる戦略**は存在しない (徹底検証済)
- **人間(友人)の52%勝率**は存在する → この edge を消費しない execution が核
- **人間の判断 × 機械の規律** の分業:
  - 人間: パターン認識・シグナル判断
  - 機械: SEQ 管理・利確/損切・BET 執行・状態監視

## GUI の物理的機能

### ✅ できること
1. Pragmatic/Evolution テーブルに自動入場
2. WebSocket リアルタイム監視 (BET 窓開閉・結果・罫線)
3. BET 執行 (座標クリック or WS 直送)
4. SEQ 進行・利確リセット・損切自動判定
5. 友人との指示通信 (Web app / Telegram / 直結)
6. 決定ログの完全記録

### ❌ できないこと
1. Canvas 描画 UI の画面読み取り (WS に来る情報で代替)
2. ゲーム結果の予測
3. BET 窓の延長 (13-18秒で判断実行まで完了必須)
4. サーバー側ルールの迂回
5. 認証なし運用

## 最大リスク: SESSION EXPIRED 問題

友人の判断は LOOK が多い → 長時間 BET せず → テーブル/セッションから追い出される → GUI は無効状態 → 友人が BET ボタン押しても空振り → 資金管理が狂う。

### 多層防御
1. **予防**: 1分ごとに最小BET ($0.20) を維持BET として投入 (session 活性化)
2. **検知**: WS切断 / DOM SESSION EXPIRED モーダル / 状態更新停止 の3重監視
3. **自動復旧**: モーダル自動dismiss + lobby から再入場 + WS再接続
4. **UI状態ランプ**: 🟢 テーブル / 🟢 WS / 🟢 BET可能 / 1つでも🔴なら BET ボタン無効化
5. **ACK確認**: BET後に WS から CLIENT_BET_ACCEPTED 受信まで待機、成否を友人画面に表示
6. **Telegram アラート**: 異常時にバックアップ通知

**絶対原則**: 友人は「打てた/打ててない」を迷わない UI。押せる時は押せる、押せない時は押せない、結果は必ずフィードバック。

## AI 学習パイプライン (Phase構成)

### Phase 1: 決定ロガー (必須・初期実装)
GUI に毎決定記録機能:
- タイムスタンプ / テーブル / 大路・珠盤路・派生罫線3種のスナップショット
- 直前10手・現在のSEQ/PNL・友人の決定(BET_P/BET_B/LOOK)・次手結果
- WS からデータ完全取得可 (Vision 不要)

### Phase 2: データ蓄積 (2-4週間)
- 友人 100-200時間プレイ → 5,000-10,000決定ポイント

### Phase 3: ルール抽出 (決定木可視化)
- 友人の判断を言語化可能なルールへ
- **1,000決定あれば実用**

### Phase 4: 機械学習 (XGBoost or 小型NN)
- 特徴量 50-100種類
- 精度 90%+ で実用
- **5,000決定で安定**

### Phase 5: LLM Vision 併用 (実験的)
- 曖昧ケース (確率40-60%) のみ Claude Sonnet/GPT-4V に問い合わせ
- 大路画像 + few-shot (友人の過去判断20件) で推論
- レイテンシ 2-5秒が BET 窓に収まるか要検証

### Phase 6: AI 自動判断モード
- 友人不在でも AI 代理
- 同席時は AI 推奨 → 友人承認
- 自動化比率を段階的に上げる

## AI Vision の使いどころ (重要な切り分け)

- ❌ 罫線認識: 不要 (WS に完璧な JSON で来る)
- ✅ **友人の判断学習**: Phase 5 の LLM 推論
- ✅ **WS外の情報**: ディーラー仕草・他プレイヤー動向・カード実画像等 (補助材料)

## ビジネスモデル

### 従来の戦略販売
- 「勝てる戦略を売る」→ 勝率50%ゲームで無理

### このシステムの販売モデル
- 「**友人の判断力を機械で再生するツール**」
- 友人の知的財産 (AI学習済みモデル) をライセンス化
- 友人にロイヤリティ配分
- ユーザーは自分の判断で使う (当たる保証はないが、平均52%の判断を受けられる)
- 法的に戦略販売より筋が良い

## 🚀 アーキテクチャ革新: ブラウザ廃止 (2026-04-17 追記)

LAPLACE (現行) は Camoufox ブラウザを24/7常駐させて iframe経由で DOM クリックし BET 実行している。これが「iframe壊死」「Camoufoxクラッシュ」「UI変更で壊れる」等の主要障害の温床。

**友人代打ちシステムならブラウザをほぼ廃止できる**。理由: Player/Banker/Tie結果、罫線、BET窓開閉、残高は全部 WebSocket (wss://dga.pragmaticplaylive.net/ws + wss://gs18.pragmaticplaylive.net/game) から JSON で来る。BETコマンドも WS (XML) で直送できる。ブラウザ必須なのは**認証トークン取得のみ**で、これは1時間ごとに15秒だけ動かせばよい。

### 新アーキテクチャ
```
VPS:
  1. Auth モジュール (cron 1h) — 短時間 Camoufox で cookie/ssid/JSESSIONID 取得 → tokens.json
  2. WS Executor (常駐 Python) — 2つのWS を維持、BET 送信、決定ロガー
  3. Web API (FastAPI) — 友人からの指示受信、状態返却

友人デバイス:
  - Web アプリ (ブラウザでOK — Stake.com ではなく**我々の UI**)
  - 大路・罫線・履歴・BETボタン
  - Tailscale/HTTPS で VPS 接続
```

### LAPLACE (現行) vs LAPLACE2 v2 (友人代打ち)
| 項目 | 現LAPLACE | LAPLACE2 v2 |
|---|---|---|
| ブラウザ | 常駐 24/7 | **1h に15秒だけ** |
| iframe 壊死 | 頻発 | **発生しない** |
| メモリ | 2-3GB | **50-100MB** |
| BET 速度 | 3-5秒 (DOM クリック) | **100ms以下** (WS直送) |
| 障害ポイント | 5層 | **1層 (WS)** |
| デプロイ | Xvfb / GPU / 大メモリ | **普通のLinux VPS** |

### 認証トークン管理
- ssid / JSESSIONID: 数時間で失効
- Stake cookies: 週単位
- → **45分ごとに token refresh ジョブ**で切れ目なく運用可
- 自動ログイン失敗時は「ユーザー手動ログイン → token 渡す」手動フォールバック

### 実装上の重要リスク
1. **BET WS コマンドのXMLフォーマット未確認** — 実BET 1回観測が必要 (ping 以外の send 例が未取得)
2. **Stake 検知回避** — pure Python で User-Agent / Origin を Camoufox と同等に揃える必要
3. **Cloudflare 強化** — 自動ログインが困難になる可能性、手動フォールバック設計必須

### 更新版ロードマップ (ブラウザ廃止前提)
1. **実BET 1回観測** (XML コマンド確定) ← ブロッカー
2. **Token 取得モジュール** (短時間Camoufox → tokens.json)
3. **WS Executor (pure Python)** 実装 — collector_pragmatic.py の延長で可能
4. **FastAPI + 友人用 Web UI** 構築
5. **決定ロガー** 組込 (AI学習データ蓄積)
6. **Session refresh ループ** (45min 間隔)
7. **少額ライブテスト**
8. (Phase AI) 学習モデル訓練 → 代理モード

### この設計の戦略的意義
- LAPLACE Evolution 版で積み上げた **「ブラウザ依存のつらみ」を根本解消**
- LAPLACE2 v2 は 友人代打ち + (将来) AI代理 の **専用設計**
- 商用配布時のユーザー体験も劇的改善 (複雑な Camoufox セットアップ不要、軽量サーバーで動く)
